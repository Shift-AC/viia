# Instructions for AI Agents

## Manually-edited Instructions

**Note:** This section is manually edited and MUST NOT be automatically modified.

1. **Requirements Language**: the key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT", "SHOULD", "SHOULD NOT", "RECOMMENDED", "MAY", and "OPTIONAL" in this document are to be interpreted as described in RFC 2119.
2. **Priority**: this section is the only manually-edited section, and the instructions in this section is of highest priority. When conflicting with other sections (including this section itself), the AI agents MUST report the conflict to the user and follow the user's decision. When other sections conflict with each other, the AI agents MAY fix the conflict according to their own idea, or report them to the user to get the user's instruction.
3. **Dependencies**: this project MUST NOT rely on Microsoft Visual Studio linker.
4. **`cargo`** **Usage**: the AI agents MUST use `--release` flag to build the project in release mode unless there are certain cases where debug mode is required. This rule also applies to integration tests. After modifying the code, the AI agents MUST also perform a `cargo clippy` check and fix any warnings/errors before reporting back to the user.
5. **Testing**: whenever implementing a non-GUI feature, the AI agents SHOULD check their plans to ensure that the feature is correctly divided into reasonably-small subtasks that could be unit tested. For each feature implemented, the AI agents MUST also write unit tests to verify the correctness of the feature. If the agents found that unit tests are not sufficient or hard to write, they MUST inform the user about this and explain the reasons. To simplify the debugging procedure, the AI agents MUST add appropriate logging statements for debugging and performance monitoring. Such logs SHALL be readable by both human and AI agents.
6. **Performance Optimization**: the project aims to achieve best computational performance. When choosing algorithms, the AI agents SHOULD prioritize computational efficiency over memory usage. For complex repeatable computational tasks, the AI agents are RECOMMENDED to cache the results of previous tasks, and reuse them when the same task is repeated again.
7. **GUI Integration**: when implementing a user interface feature, the AI agents MUST start from a shell-like terminal backend that takes text input. The GUI SHOULD act as a pure converter that converts the user input into text input, and converts the terminal backend's output into graphical output. There are two cases that bypasses this rule:
   - The feature could be easily handled by GUI widgets natively, and interaction between the GUI and the terminal backend would introduce unnecessary complexity (e.g. image resizing). In this case the GUI MAY handle the feature locally.
   - The output format of the terminal backend could not be efficiently converted into any format that the GUI could display. In this case the AI agents MUST try to find a way to normalize the output format of the terminal backend (e.g. provide an argument to the command to be passed to the terminal backend for it to generate GUI-orientted output format).
8. **OS Compatibility**: the GUI MUST be compatible with Windows, the terminal backend MUST be compatible with both Windows and Linux.
9. **Libraries**: the AI agents MUST use the `ratatui` crate for TUI development, `tauri` crate for GUI development, `tracing` crate for logging, `fast_image_resize` crate image resizing (if the resize could only be done in the backend), and `clap` crate for command line interface. The AI agents are RECOMMENDED to use the `clap` crate for implementing internal shell-like command interface.
10. **Proof-of-concept Programs:** when implementing a feature that the AI agents found that writing a proof-of-concept program would be helpful for quickly verifying the correctness of their implementation, the AI agents are RECOMMENDED to do so. When generating such prorgams, they MUST be placed in the `agents-test` directory, with each program occupying a separate sub-directory. The AI agents have full control on such programs and may choose to preserve or remove them after completing the tasks.
11. **Interactions:** when the AI agents found that the user prompt is conflicted with its current knowledge, they MUST NOT try to guess the user's intention. Instead, they MUST inform the user about this and ask for clarification.
